# 修复说明 - 锁机时段问题

## 问题描述

### 问题 1：时间拦截失效
- **现象**：设置 23:40～04:00 拦截，但早上 08:30 打开 Bilibili 仍被拦截
- **期望**：04:00 后应自动解锁，08:30 不应被拦截

### 问题 2：息屏后仍在工作
- **现象**：日志显示息屏后一直都在工作
- **期望**：屏幕熄灭时应暂停监控，减少日志输出

---

## 根本原因

### 问题 1 原因
1. **解锁定时任务可能未触发**：`SchedulerManager` 设置的 04:00 解锁闹钟可能因系统省电策略未触发
2. **缺少自动保护机制**：`MonitorService` 每 5 秒检查时间段，但即使发现不在锁机时段，也只是设置 `shouldBeLocked=false`，**没有自动停止锁机服务**
3. **结果**：如果 04:00 的解锁广播未触发，`is_lock_active` 标志保持为 `true`，导致继续拦截

### 问题 2 原因
- `MonitorAccessibilityService.startPeriodicCheck()` 每 10 秒调用 `updateLockPeriod()`，**无论屏幕是否亮起都输出日志**
- 虽然实际拦截逻辑检查了屏幕状态（`checkCurrentApp` 中检查 `isScreenOn`），但日志输出让你误以为一直在工作

---

## 修复方案

### 修复 1：添加自动停止保护机制
**文件**：`MonitorService.kt`

在 `updateLockPeriod()` 方法中添加：
```kotlin
// 关键修复：如果不在锁机时段但 isLockActive 仍为 true，自动停止锁机服务
if (!isInTimePeriod && isLockActive && !testMode) {
    Log.w(TAG, "⚠️ 检测到不在锁机时段但服务仍激活，自动停止锁机服务")
    
    // 自动关闭锁机服务
    prefs.edit().putBoolean(KEY_LOCK_ACTIVE, false).apply()
    
    // 停止前台服务
    stopForeground(STOP_FOREGROUND_REMOVE)
    stopSelf()
    
    Log.d(TAG, "✅ 已自动停止锁机服务")
    return@launch
}
```

**效果**：即使 04:00 的解锁广播未触发，`MonitorService` 在 05 秒内检测到当前时间不在锁机时段后，会自动停止服务并清除 `is_lock_active` 标志。

### 修复 2：息屏时不输出日志
**文件**：`MonitorAccessibilityService.kt`

修改 `startPeriodicCheck()` 方法：
```kotlin
// 只在屏幕亮起时输出日志（最多每分钟一次）
val isScreenOn = withContext(Dispatchers.Main) {
    val powerManager = getSystemService(Context.POWER_SERVICE) as android.os.PowerManager
    powerManager.isInteractive
}

val now = System.currentTimeMillis()
if (isScreenOn && now - lastLogTime > 60000) {
    // 输出日志
    lastLogTime = now
}
```

**效果**：息屏时不输出日志，避免误以为一直在工作。

### 修复 3：增加调试日志
**文件**：`MonitorService.kt` 和 `MonitorAccessibilityService.kt`

添加详细的时间计算日志：
```kotlin
Log.v(TAG, "⏰ 时间检查：当前=$currentMinutes (HH:MM), " +
    "锁机=$lockMinutes-$unlockMinutes (HH:MM-HH:MM), " +
    "在时段内=$isInTimePeriod")
```

**效果**：方便排查时间计算是否正确。

## 额外建议

### 建议 1：检查系统闹钟权限
确保应用有权限设置精确闹钟：
- 设置 → 应用 → 专注锁机 → 电池 → 允许后台活动
- 设置 → 应用 → 专注锁机 → 权限 → 闹钟和提醒 → 允许

### 建议 2：添加解锁时段自动启动保护
如果问题持续，可以在 `MainActivity.kt` 中添加：
- 启动时检查当前时间，如果不在锁机时段自动停止服务
- 添加"一键重置"按钮，手动清除异常状态

### 建议 3：使用 WorkManager 替代 AlarmManager
`AlarmManager` 在 Android 12+ 可能受限制，考虑迁移到 `WorkManager`：
```kotlin
WorkManager.enqueueUniquePeriodicWork(
    "unlock_work",
    ExistingPeriodicWorkPolicy.REPLACE,
    PeriodicWorkRequestBuilder<UnlockWorker>(15, TimeUnit.MINUTES)
        .setConstraints(Constraints.Builder().build())
        .build()
)
```
